# SAM-DAM Berry Segmentation

This repository contains the cleaned local code for berry and plant segmentation experiments based on YOLO detectors, SAM-style segmentation models, IoU evaluation, and depth-based 3D visualization.

Paper link: https://www.sciencedirect.com/science/article/pii/S0168169925006076

## Repository Layout

- `samdam/`: shared segmentation, prompt-selection, overlap, IoU, saving, and 3D helper functions.
- `scripts/`: runnable benchmark and utility scripts.
- `SAMs/`: minimal vendored EfficientSAM and MobileSAM source needed by the benchmark code. Model weights are not included.
- `examples/GNV_benchmark_data_coco/`: small COCO-format sample dataset copied from the original local project.
- `models_wts/`: expected location for local model weights. This directory is intentionally ignored by git.
- `outputs/`: default output directory. Generated files are intentionally ignored by git.

## Setup

```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

The benchmark expects model weights in `models_wts/` unless `SAMDAM_MODEL_DIR` is set:

```text
models_wts/
  SAMDAM_fruit.pt
  SAMDAM_plant.pt
  sam_vit_h_4b8939.pth
  l2.pt
  mobile_sam.pt
  sam_vit_h.pt
```

Additional variant-specific weights:

```text
SAMs/EfficientSAM/weights/
  efficient_sam_vitt.pt
  efficient_sam_vits.pt

SAMs/MobileSAM/MobileSAMv2/PromptGuidedDecoder/
  Prompt_guided_Mask_Decoder.pt
```

## Run

Run the default SAM-DAM benchmark configuration on the included sample data:

```bash
bash scripts/run_samdam_benchmark.sh
```

Or call the benchmark directly:

```bash
python scripts/run_samdam_benchmark.py \
  --pointcase G \
  --method 2p_8 \
  --pre_mask with_pre_mask \
  --box_type overlapbox \
  --use_preliminary_masks true \
  --use_exclusive_points true \
  --base_dir outputs/GNV_benchmark_results \
  --image_folder examples/GNV_benchmark_data_coco \
  --annotations_path examples/GNV_benchmark_data_coco/annotations.json \
  --iou_output_dir outputs/iou_results/GNV \
  --method_name 2p_8 \
  --which_sam SAM \
  --which_efficient_sam EfficientSAM_s \
  --mobv2_encoder_type efficientvit_l2 \
  --newmethod true
```

Classical baseline segmentation and IoU utilities. Pass `--bbox_folder` to a folder of YOLO-format bounding boxes, for example a `yolo_bboxes/` folder generated by a benchmark run:

```bash
python scripts/run_classical_segmentation.py --bbox_folder <path_to_yolo_bboxes>
python scripts/evaluate_classical_iou.py --timestamp <run_timestamp>
```

Depth-image generation utility:

```bash
python scripts/generate_depth_images.py \
  --input_dir examples/GNV_benchmark_data_coco \
  --output_dir outputs/depth/GLPN
```

## Notes

Large datasets, model weights, generated masks, plots, and IoU outputs were left out of the GitHub repository so the codebase stays portable. The original local project remains unchanged.

The vendored EfficientSAM and MobileSAM code keeps its upstream license files in `SAMs/EfficientSAM/LICENSE` and `SAMs/MobileSAM/LICENSE`. Add a project-level license before formal public release if needed.
